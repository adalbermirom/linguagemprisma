funcao es.abra(caminho, modo)

Abre um arquivo e retorna um manipulador de arquivo.

Parâmetros:

{string} - Caminho do arquivo a ser aberto.
{string} - Modo de abertura ("leitura", "escrita", "adicao", etc).

Modos válidos:

  • leitura
  • escrita
  • leiturabin
  • escritabin
  • leitura+
  • escrita+
  • leitura+bin
  • escrita+bin
  • adicao (ou adição)
  • adicaobin (ou adiçãobin)
  • adicao+bin (ou adição+bin)

Modos também válidos mas curtos:

  • l - leitura
  • e - escrita
  • lb - leiturabin
  • eb - escritabin
  • l+ - leitura+
  • e+ - escrita+
  • l+b - leitura+bin
  • e+b - escrita+bin
  • a - adição
  • ab - adicaobin (ou adiçãobin)
  • a+b - adicao+bin (ou adição+bin)

Obs.: Os modos em inglês também são aceitos 'r', 'w', 'a', 'r+' etc.

Retornos:

{arquivo} - Manipulador de arquivo ou nulo em caso de erro.
{string} - String de erro se houver.
📋
 
local arq, err = es.abra("dados.txt", "leitura")
se nao arq entao erro(err); fim

funcao es.leia(...)

Lê dados da entrada padrão (geralmente o teclado) ou de um arquivo.

Parâmetros:

{string} - Opção de leitura:
  • "*n" - lê um número
  • "*l" - lê uma linha (sem o caractere de nova linha)
  • "*L" - lê uma linha (com o caractere de nova linha)
  • "*a" - lê o arquivo inteiro
  • "*t" - lê o arquivo inteiro (sinônimo de "*a")

Retornos:

{string|numero|nulo} - Dados lidos, ou nulo em caso de erro ou fim de arquivo.
📋
 
local num = es.leia("*n");
imprima(tipo(num));

funcao arquivo:busque(offset, origem)

Move o ponteiro de leitura/escrita no arquivo.

Parâmetros:

{numero} - Posição (offset) a mover.
{string} - Origem:
  • "def" - início do arquivo (SEEK_SET)

Retornos:

{numero|nil} - Nova posição ou nil em caso de erro.
📋
